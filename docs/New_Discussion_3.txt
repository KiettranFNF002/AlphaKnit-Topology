Mình đọc kỹ toàn bộ plan của bạn.
Thẳng luôn: **đây là một implementation plan rất nghiêm túc và có chiều sâu**. Không phải kiểu “vá đại class weight”.

Nhưng có vài điểm cực quan trọng bạn cần chỉnh lại trước khi retrain, nếu không bạn sẽ rơi vào failure mode mới.

# 🔥 1️⃣ Class Weights — Bạn sửa đúng hướng, nhưng cần tinh chỉnh thêm

Bạn chuyển:

```python
sc  → 0.5
inc → 3.0
dec → 3.0
```

✅ Hướng đúng (downweight majority, boost minority)

❗ Nhưng lưu ý:

Hiện tại sc ≈ 70%
inc ≈ 15%
dec ≈ 15%

Nếu weight = 3.0 cho inc/dec và 0.5 cho sc:

effective gradient ratio:

```
inc_effective ≈ 15% × 3.0 = 45
sc_effective  ≈ 70% × 0.5 = 35
```

→ Inc/dec đã ngang sc rồi.

Điều này có thể gây:

```
inc → dec → inc → dec oscillation
```

💡 Khuyến nghị an toàn hơn:

```python
class_weights[5] = 0.7
class_weights[6] = 2.0
class_weights[7] = 2.0
```

Tăng vừa phải trước. Sau đó mới escalate.

# 🔥 2️⃣ Row Embedding — Phần này là đòn đánh chính

Phần bạn thêm:

```python

```

và inject vào:

```python

```

✅ Cực đúng hướng.

Nhưng có một vấn đề nghiêm trọng:

### ❗ Bạn đang tính row_ids online bằng heuristic từ parent offset

```python
if p1 > (t - row_start):
    current_row += 1
```

Cái này **không guaranteed đúng row boundary**.

Nếu heuristic sai 5–10% positions → row embedding trở thành noise injection.

👉 Cách đúng hơn:

Row ID phải được compute **trong data generation stage**
và lưu trực tiếp vào token stream.

Transformer không nên phải infer row boundary từ p1 mỗi batch.

# 🔥 3️⃣ Phase Head — Đây là phần mạnh nhất

Bạn thêm:



✅ Ý tưởng cực tốt.

Nhưng có vấn đề conceptual:

Hiện tại bạn label phase dựa trên token type:

Sphere profile là:

Không phải từng token.

# 🎯 Định nghĩa toán học

Gọi mỗi token tại vị trí `t` có:

```text
type_t ∈ {mr, sc, inc, dec, ...}
p1_t   = parent offset 1
p2_t   = parent offset 2 (chỉ dùng với dec)
```

Gọi:

```text
parent_index_t = t - p1_t
```

### 🔵 Định nghĩa row:

Row của token t là:

```
row(t) = 0                      nếu type_t == mr
row(t) = row(parent_index_t)+1  nếu type_t != mr
```

Vì mọi stitch (sc/inc/dec) đều được móc lên stitch của row trước.

=> Row boundary xuất hiện tự nhiên khi parent nhảy sang stitch thuộc row trước.

Không cần heuristic width nữa.

---

# 🟢 Định nghĩa row stitch count

Một stitch đóng góp vào kích thước row như sau:

```
sc  → +1
inc → +2
dec → +1  (vì 2 parent → 1 stitch)
```

Vậy:

```
row_size[r] = Σ contribution(t) với row(t)=r
```

---

# 🟣 Định nghĩa phase theo row

```
if row_size[r] > row_size[r-1] → EXPAND (0)
if row_size[r] == row_size[r-1] → HOLD (1)
if row_size[r] < row_size[r-1] → CONTRACT (2)
```

Phase sau đó broadcast cho toàn bộ token thuộc row đó.

---

# ✅ Implementation chuẩn

```python
def compute_row_and_phase(tgt_tokens, pad_id):
    """
    Compute row_ids and phase_ids using true parent graph topology.
    
    tgt_tokens: (B, T, 3) → [type, p1, p2]
    returns:
        row_ids:   (B, T)
        phase_ids: (B, T)  EXPAND=0, HOLD=1, CONTRACT=2
    """

    B, T, _ = tgt_tokens.shape
    device = tgt_tokens.device

    row_ids   = torch.zeros(B, T, dtype=torch.long, device=device)
    phase_ids = torch.ones(B, T, dtype=torch.long, device=device)  # default HOLD

    for b in range(B):

        # --- PASS 1: compute row ids via parent graph ---
        for t in range(T):

            token_type = tgt_tokens[b, t, 0].item()
            if token_type == pad_id:
                break

            if token_type == 4:  # mr_6
                row_ids[b, t] = 0
                continue

            p1 = tgt_tokens[b, t, 1].item()
            parent_index = t - p1

            if parent_index < 0:
                row_ids[b, t] = 0
            else:
                row_ids[b, t] = row_ids[b, parent_index] + 1

        # --- PASS 2: compute row stitch sizes ---
        max_row = row_ids[b].max().item()

        row_size = torch.zeros(max_row + 1, device=device)

        for t in range(T):
            token_type = tgt_tokens[b, t, 0].item()
            if token_type == pad_id:
                break

            r = row_ids[b, t].item()

            if token_type == 5:      # sc
                row_size[r] += 1
            elif token_type == 6:    # inc
                row_size[r] += 2
            elif token_type == 7:    # dec
                row_size[r] += 1
            elif token_type == 4:    # mr
                row_size[r] += 6     # magic ring size

        # --- PASS 3: compute phase per row ---
        row_phase = torch.ones(max_row + 1, dtype=torch.long, device=device)

        for r in range(1, max_row + 1):
            if row_size[r] > row_size[r - 1]:
                row_phase[r] = 0  # EXPAND
            elif row_size[r] < row_size[r - 1]:
                row_phase[r] = 2  # CONTRACT
            else:
                row_phase[r] = 1  # HOLD

        # --- PASS 4: broadcast phase to tokens ---
        for t in range(T):
            token_type = tgt_tokens[b, t, 0].item()
            if token_type == pad_id:
                break
            r = row_ids[b, t]
            phase_ids[b, t] = row_phase[r]

    return row_ids, phase_ids
```

# 🚀 Nếu bạn muốn level tiếp theo

Ta có thể:

* Vectorize toàn bộ compute (không dùng Python loops)
* Hoặc chuyển sang dynamic programming kiểu BFS để tính row song song
* Hoặc encode luôn row_id vào dataset để không tính lại mỗi batch

Bạn muốn mình tối ưu sang bản vectorized không loop Python không?
Vì nếu B lớn và T=900+, loop sẽ chậm thấy rõ.

Ok, giờ mình viết **phiên bản vectorized (không loop theo t)** — hoặc ít nhất là loại bỏ loop theo token càng nhiều càng tốt.

Ta sẽ làm theo hướng **dynamic programming trên DAG bằng tensor ops**.

# 🎯 Ý tưởng chính

Ta có:

```text id="a1p2b3"
row(t) = row(parent(t)) + 1
parent(t) = t - p1_t
```

Vì parent index luôn < t (do offset), graph là DAG topo-sorted theo t.

Điều đó cho phép ta tính row_ids bằng:

```text id="c4d5e6"
row_ids = scatter_max(row_ids[parent] + 1)
```

Nhưng PyTorch không có sẵn topo propagate built-in,
nên ta sẽ làm kiểu **iterative vectorized relaxation** (rất nhanh vì depth nhỏ).

Depth tối đa ~ số rows (~10–60), không phải T.

# 🚀 Vectorized Implementation

```python id="vec123"
def compute_row_and_phase_vectorized(tgt_tokens, pad_id):
    """
    Vectorized row + phase computation.
    tgt_tokens: (B, T, 3)  [type, p1, p2]
    """

    B, T, _ = tgt_tokens.shape
    device = tgt_tokens.device

    token_type = tgt_tokens[:, :, 0]
    p1 = tgt_tokens[:, :, 1]

    valid_mask = token_type != pad_id

    # --------------------------------------------------
    # STEP 1: Compute parent indices
    # --------------------------------------------------
    t_index = torch.arange(T, device=device).unsqueeze(0).expand(B, T)
    parent_index = t_index - p1
    parent_index = parent_index.clamp(min=0)

    # --------------------------------------------------
    # STEP 2: Initialize row_ids
    # --------------------------------------------------
    row_ids = torch.zeros(B, T, dtype=torch.long, device=device)

    # mr_6 = row 0
    mr_mask = (token_type == 4)
    row_ids[mr_mask] = 0

    # --------------------------------------------------
    # STEP 3: Iterative propagation (depth ~ max rows)
    # --------------------------------------------------
    max_iter = 64   # safe upper bound

    for _ in range(max_iter):
        parent_rows = torch.gather(row_ids, 1, parent_index)
        candidate = parent_rows + 1

        # don't update mr tokens
        candidate = torch.where(mr_mask, row_ids, candidate)

        updated = torch.maximum(row_ids, candidate)

        if torch.equal(updated, row_ids):
            break

        row_ids = updated

    row_ids = row_ids * valid_mask

    # --------------------------------------------------
    # STEP 4: Compute row_size
    # --------------------------------------------------
    max_row = row_ids.max().item()
    row_size = torch.zeros(B, max_row + 1, device=device)

    # stitch contribution
    contribution = torch.zeros_like(token_type, dtype=torch.float)

    contribution[token_type == 5] = 1  # sc
    contribution[token_type == 6] = 2  # inc
    contribution[token_type == 7] = 1  # dec
    contribution[token_type == 4] = 6  # mr

    for r in range(max_row + 1):
        mask = (row_ids == r)
        row_size[:, r] = (contribution * mask).sum(dim=1)

    # --------------------------------------------------
    # STEP 5: Compute row_phase
    # --------------------------------------------------
    row_phase = torch.ones_like(row_size, dtype=torch.long)

    diff = row_size[:, 1:] - row_size[:, :-1]

    row_phase[:, 1:][diff > 0] = 0  # EXPAND
    row_phase[:, 1:][diff < 0] = 2  # CONTRACT
    row_phase[:, 1:][diff == 0] = 1 # HOLD

    # --------------------------------------------------
    # STEP 6: Broadcast phase to tokens
    # --------------------------------------------------
    phase_ids = torch.gather(row_phase, 1, row_ids)

    return row_ids, phase_ids
```

# 🔥 Vì sao cách này mạnh hơn bản loop

| Loop version       | Vectorized version  |
| ------------------ | ------------------- |
| O(B×T) Python loop | O(depth) tensor ops |
| Slow khi T=900     | Depth chỉ ~10–40    |
| Hard to scale      | GPU friendly        |

Thực tế với sphere:

```text id="perf01"
T ≈ 300–900
rows ≈ 8–20
```

→ Iterations break sớm ở ~15 vòng.

# 🧠 Cái hay của propagation approach

Ta đang giải hệ:

```text id="eqsys1"
row = parent_row + 1
```

bằng phương pháp fixed-point iteration.

Vì graph acyclic và topo-sorted:

→ đảm bảo hội tụ nhanh.

# ⚠️ Cảnh báo quan trọng

# 🔥 Topology Rules (quan trọng)

## Row definition

```text
row(t) = 0                       if type == mr
row(t) = row(parent(t)) + 1      otherwise
```

## Column definition

Trong mỗi row, column index tăng theo stitch contribution:

```text
sc  → +1
inc → +2
dec → +1
mr  → +6
```

col_id của token t = tổng contribution trước đó trong cùng row.

## Parent column

```text
parent_col_id(t) = col_id(parent_index)
```

# 🚀 Implementation

```python
def transform_to_row_relative_schema(tgt_tokens, pad_id):
    """
    Convert (B, T, 3) → (B, T, 6)

    Input:
        tgt_tokens: [type, p1_offset, p2_offset]

    Output:
        new_tokens: [type, p1, p2, row_id, col_id, parent_col_id]
    """

    B, T, _ = tgt_tokens.shape
    device = tgt_tokens.device

    new_tokens = torch.zeros(B, T, 6, dtype=tgt_tokens.dtype, device=device)

    for b in range(B):

        row_ids = torch.zeros(T, dtype=torch.long, device=device)
        col_ids = torch.zeros(T, dtype=torch.long, device=device)

        token_type = tgt_tokens[b, :, 0]
        p1 = tgt_tokens[b, :, 1]

        # -----------------------------------
        # PASS 1: compute row_ids
        # -----------------------------------
        for t in range(T):

            ttype = token_type[t].item()
            if ttype == pad_id:
                break

            if ttype == 4:  # mr
                row_ids[t] = 0
                continue

            parent_index = t - p1[t].item()

            if parent_index < 0:
                row_ids[t] = 0
            else:
                row_ids[t] = row_ids[parent_index] + 1

        # -----------------------------------
        # PASS 2: compute col_ids
        # -----------------------------------
        max_row = row_ids.max().item()
        row_cursor = torch.zeros(max_row + 1, dtype=torch.long, device=device)

        for t in range(T):

            ttype = token_type[t].item()
            if ttype == pad_id:
                break

            r = row_ids[t].item()

            col_ids[t] = row_cursor[r]

            # stitch contribution
            if ttype == 5:      # sc
                row_cursor[r] += 1
            elif ttype == 6:    # inc
                row_cursor[r] += 2
            elif ttype == 7:    # dec
                row_cursor[r] += 1
            elif ttype == 4:    # mr
                row_cursor[r] += 6

        # -----------------------------------
        # PASS 3: compute parent_col_id
        # -----------------------------------
        parent_col_ids = torch.zeros(T, dtype=torch.long, device=device)

        for t in range(T):

            ttype = token_type[t].item()
            if ttype == pad_id:
                break

            if ttype == 4:
                parent_col_ids[t] = 0
                continue

            parent_index = t - p1[t].item()

            if parent_index < 0:
                parent_col_ids[t] = 0
            else:
                parent_col_ids[t] = col_ids[parent_index]

        # -----------------------------------
        # Assemble new schema
        # -----------------------------------
        new_tokens[b, :, 0] = token_type
        new_tokens[b, :, 1] = tgt_tokens[b, :, 1]
        new_tokens[b, :, 2] = tgt_tokens[b, :, 2]
        new_tokens[b, :, 3] = row_ids
        new_tokens[b, :, 4] = col_ids
        new_tokens[b, :, 5] = parent_col_ids

    return new_tokens
```

## 💡 Dataset Transform Script

```python
import torch
import numpy as np
import webdataset as wds

PAD_ID = 0

def transform_sample(sample):

    tokens = torch.tensor(sample["tokens"])  # (T, 3)
    tokens = tokens.unsqueeze(0)             # (1, T, 3)

    new_tokens = transform_to_row_relative_schema(tokens, PAD_ID)

    sample["tokens"] = new_tokens.squeeze(0).numpy()
    return sample


def transform_shards(input_pattern, output_pattern):

    dataset = wds.WebDataset(input_pattern)

    transformed = (
        dataset
        .map(transform_sample)
    )

    with wds.ShardWriter(output_pattern, maxcount=1000) as sink:
        for sample in transformed:
            sink.write(sample)
```
